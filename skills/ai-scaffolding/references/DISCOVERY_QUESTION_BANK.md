# Discovery Question Bank

Use this bank to keep questioning focused and proportional to project maturity.

## First pass

Ask these only if not inferable from repository evidence:

1. What stack should this scaffolding prioritize first?
2. Is the project greenfield or already in active development?
3. Which quality priorities matter most now (delivery speed, reliability, security, maintainability)?

## Greenfield mode (ask more)

Use up to 8 questions total when context is sparse.

Suggested areas:

1. Primary stack and framework.
2. Architecture target (monolith, modular monolith, microservices).
3. Team size and collaboration model.
4. Testing depth expected at bootstrap.
5. Deployment environment constraints.
6. Security/compliance constraints.
7. Scope of initial rules (strict or lightweight).
8. Initial subagent focus areas.

## In-progress mode (ask less)

Use up to 3 questions total.

Prioritize:

1. Which parts of current context are failing or missing?
2. What should remain untouched to avoid workflow disruption?
3. Which gaps are highest priority right now?

## Question quality rules

- Ask only decision-critical questions.
- Avoid yes/no questions if a short multiple-choice question gives clearer signal.
- Bundle related questions in one message when possible.
- Stop asking once confidence is sufficient for concrete prompts.
